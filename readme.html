Do the following to deploy the application on your web server:

  • Unzip the downloaded app.
  • Add a proxy to your app. Refer to Use proxy topic in Web AppBuilder help.
  • Add and register the app. Refer to  Add apps  and Register your app topic in ArcGIS Online help or Portal for ArcGIS help.
  • Copy and paste the AppID to the appId property in the config.json file. Save the file.
  • Deploy the app on your web server.

Optionally, do the following to customize the application by editing JSON files directly:

  • Add custom widgets
    1. Put your custom widget in widgets folder.
    2. Add widget into config.json.
      The example below uses the Demo widget. Add it into config.json file

      {
          "name": "Demo",
          "label": "Demo",
          "uri": "widgets/Demo/Widget",
          "index": 13,
          "id": "widgets/Demo/Widget_1"
      },

    3. Open widgets/widgets-manifest.json, add a new pair of key/value.

      Note: If the app is downloaded from Web AppBuilder for ArcGIS (Developer Edition), skip this step.

      The key is widget’s uri, the value is the content from your widget’s manifest.json.

      The example below uses the Demo widget. Add "widgets/Demo/Widget" key and copy its manifest.json content to the widgets-manifest.json here.

      "widgets/Demo/Widget": {
          "name": "Demo",
          "platform": "HTML",
          "version": "0.0.1",
          "author": "Esri R&D Center Beijing",
          "description": "This is the widget used in developer guide",
          "copyright": "",
          "license": "http://www.apache.org/licenses/LICENSE-2.0"
      },

  • Configure app and widgets. Refer to JSON configuration reference topic in Web AppBuilder help.
  • Reference preferred ArcGIS API for JavaScript. Refer to Change URL of ArcGIS API for JavaScript topic in Web AppBuilder help.
